# Codex 编程学习指南

给技术小白的一份 AI 编程协作手册。

如果你不会编程，但想用 Codex 做网页、工具或复杂系统，这份指南会教你如何从一个模糊想法开始，逐步完成：

- 需求梳理
- 原型设计
- 代码实现
- 功能测试
- 部署上线
- 项目归档

## 适合谁

- 不会编程，但想做自己的网页或工具
- 想学习如何和 Codex 协作的人
- 想把想法拆成可执行项目的人
- 想了解网页、系统、部署和归档基础的人

## 你会学到什么

1. 如何向 Codex 描述项目
2. 如何拆 MVP
3. 如何做网页原型
4. 如何使用对话模板
5. 如何验收功能
6. 如何部署和归档项目

## 内容结构

- 快速入门：先分清你和 Codex 各自负责什么
- 标准工作流：从目标、MVP、需求到开发、测试和归档
- 技术基础：HTML、CSS、JavaScript、本地服务、浏览器验收
- 对话模板：项目启动、需求梳理、开发执行、Bug 排查、测试验收
- 复杂系统与案例：如何分阶段做复杂系统
- 验收部署归档：如何把项目稳稳收口

## 为什么做这份指南

很多人第一次使用 AI 编程工具时，容易直接说“帮我做个系统”。这样很容易让项目范围失控，也很难验收结果。

这份指南的目标是把“和 Codex 协作做项目”变成一套更清楚的流程：

```text
想法 -> 需求 -> 原型 -> 代码 -> 测试 -> 部署 -> 归档
```

你不需要一开始就懂所有代码。你需要先学会把目标说清楚，和 Codex 一起拆小任务、做原型、测结果，再把项目资料整理好。

## 推荐用法

1. 先阅读桌面版或手机版。
2. 复制页面里的项目启动模板。
3. 把模板里的方括号替换成自己的项目情况。
4. 发给 Codex，让它先拆需求和 MVP。
5. 每完成一个阶段，都让 Codex 写测试记录和阶段总结。
